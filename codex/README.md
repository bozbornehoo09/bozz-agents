# Using bozz-agents with Codex

Codex consumes the same canonical `SKILL.md` files as other compatible agents.
No skill conversion is needed. Codex has its own plugin manifest and marketplace
metadata in addition to its skill discovery locations.

## Where Codex looks for skills

Codex discovers skills, in precedence order:

| Scope | Path |
|---|---|
| Repository | `.agents/skills/` (from cwd up to repo root) |
| User | `$HOME/.agents/skills/` |
| Admin | `/etc/codex/skills/` |

## Install as a plugin

```bash
codex plugin marketplace add bozbornehoo09/bozz-agents
codex plugin add bozz-agents@bozz-agents
```

The Codex manifest at `.codex-plugin/plugin.json` points directly at the
tool-neutral `skills/` tree. This is the preferred distribution path.

## Install skills directly

This repo's `.agents/skills/` is **generated by CI** from the canonical `skills/`
tree (see the main README, "How the skills are distributed"), so a clone already
contains a Codex-ready directory. Surface it to Codex by scope:

```bash
# User scope — available in every repo:
mkdir -p ~/.agents/skills
ln -s /absolute/path/to/bozz-agents/.agents/skills/* ~/.agents/skills/

# Or vendor a subset into one repo:
cp -R /absolute/path/to/bozz-agents/.agents/skills/docs-review .agents/skills/
```

(Symlinks keep you current with `git pull`; copies pin a snapshot. Point direct
skill discovery at the generated `.agents/skills/`; plugin packaging points at
the canonical `skills/` tree.)

## Invoke

- Explicitly: `/skills`, or type `$` and mention a skill (e.g. `$docs-review`).
- Implicitly: Codex auto-selects a skill when the task matches its
  `description`.

## Notes & expansion seams

- **Per-skill `references/`** travel with each skill folder, so a single skill
  copied on its own stays self-contained — the briefs come with it.
- **`agents/openai.yaml`** (optional, per skill) is where Codex-specific skill
  configuration would go if you need it. None of the skills here require it
  today; add one inside a skill folder only if a Codex-only behavior is needed.
- **Orientation:** Codex reads `AGENTS.md` (this repo has one at the root). The
  `update-orientation` skill resolves the consuming project's declared
  orientation source and can maintain `AGENTS.md`, `CLAUDE.md`, or a neutral
  source that generates both.
- This adapter is intentionally thin: the skills are the source of truth, Codex
  is one consumer. Adding another tool (Cursor `.cursor/skills/`, Gemini, etc.)
  is the same move — point its skills directory at `skills/`.
